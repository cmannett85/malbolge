Detailed Description

template<bool IsConstant>
class malbolge::virtual_memory::iterator_generic< IsConstant >

Iterator class.

Following the rules of Malbolge's vmem (https://en.wikipedia.org/wiki/Malbolge#Memory), incrementing pointers past the end wraps them back round to zero. This of course means that an end() will never be reached, so loops like this:

auto vmem = virtual_memory(..);
for (auto& t : vmem) {
...
}

Will never end! They need to be explicitly broken out of, but that should only happen when a Malbolge program ends.

Note
This class cannot be copied, but can be moved
