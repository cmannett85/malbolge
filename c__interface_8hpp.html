Detailed Description

Exposes a C89 interface to the important parts of the library.

A C interface makes accessing functionality from WASM or creating bindings to other languages much easier. It is not threadsafe.

◆ malbolge_vcpu_input()

int malbolge_vcpu_input ( malbolge_virtual_cpu  vcpu,
const char *  buffer,
unsigned int  size 
)

Passes buffer to vcpu to use as user input.

This request is ignored if vcpu was created with the use_cin argument set to true.

buffer is copied into an internal queue, so this can be called before vcpu is ready to receive user input. As the data is copied, you need to free buffer manually (assuming it was malloc-ed).

Parameters
vcpuVirtual CPU handle returned from malbolge_create_vcpu(malbolge_virtual_memory)
bufferInput data
sizeSize of buffer
Returns
  • MALBOLGE_ERR_SUCCESS for success
  • MALBOLGE_ERR_NULL_ARG if vcpu or buffer is NULL
  • MALBOLGE_ERR_CIN_OR_STOPPED if vcpu is set to use cin, or already stopped
