◆ add_breakpoint()

void malbolge::virtual_cpu::add_breakpoint ( math::ternary  address,
std::size_t  ignore_count = 0 
)

Adds a breakpoint to the program.

If another breakpoint is already at the given address, it is replaced. Breakpoints can be added to the program whilst in any non-STOPPED state, but results can be unpredicatable if the program is running.

The breakpoint-hit signal is fired when a breakpoint is reached. The signal is fired when then code pointer reaches the address i.e. before the instruction is executed, so you need to step to see the result of the instruction execution.

Parameters
addressAddress to attach a breakpoint
ignore_countNumber of times the breakpoint is hit before the breakpoint_hit_signal_type signal is fired
Exceptions
execution_exceptionThrown if backend has been destroyed, usually as a result of use-after-move
