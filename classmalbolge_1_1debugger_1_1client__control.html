Detailed Description

Debugger interface to the vCPU.

Allows the attaching of breakpoints and execution control of a program running on a vCPU.

This class can be moved, but not copied.

template<typename vCPU >
malbolge::debugger::client_control::client_control ( vCPU &  vcpu)
inlineexplicit

Constructor.

The only requirement that a vCPU has is to have a confiqure_debugger method which accepts a running state callback and a step data callback, and returns a populated vcpu_control instance. See virtual_cpu::configure_debugger(running_callback_type, step_data_callback_type) for an example.

Once attached the lifetimes of the vCPU and debugger are tied, as they share callbacks that call into each other.
