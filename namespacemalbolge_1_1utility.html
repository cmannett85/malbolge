template<typename T >
T malbolge::utility::from_chars ( std::string_view  str)

A simple wrapper over std::from_chars for common scenarios.

The formatting is determined by the prefix:

  • 0x or 0X is base-16 or std::chars_fmt::hex
  • 0 is base-8, integers only
  • t is trit, ternary only Scientific or fixed exponent for floating point is determined by the content of str. Any leading '+' character is stripped.

Iterates over tuple and calls f on each element.

auto t = std::tuple{"hello"s, 42.5, 9};
tuple_iterator([](auto i, auto&& v) {
std::cout << i << ": " << v << std::endl;
}, t);
// 0: hello
// 1: 42.5
// 2: 9

The signature of f must be equivalent to:

template<typename Tuple , typename F >
constexpr auto malbolge::utility::tuple_type_iterator ( F &&  f) -> std::enable_if_t<traits::is_tuple_like_v<std::decay_t<Tuple>>>
constexpr

Iterates over tuple and passes a nullptr pointer of the tuple element to f.

We pass a pointer to f because C++20 template lambda support isn't available in the Clang version used by Emscripten.

Using Tuple = std::tuple<std::string, double, int>;
tuple_iterator<Tuple>([](auto i, auto ptr) {
using Arg = std::remove_pointer_t<decltype(ptr)>;
std::cout << i << ": " << boost::core::demangle(typeid(Arg).name())
<< std::endl;
}, t);
// 0: std::string
// 1: double
// 2: int

The signature of f must be equivalent to:

template <typename T>
void f(std::size_t is, T* ptr);
